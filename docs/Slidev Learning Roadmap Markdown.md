# Slidev完全習得ガイド：実務3年エンジニア向け5日間学習プラン

中上級Webアプリケーションエンジニア向けのSlidev基礎学習について、効率的な学習戦略と実践的な習得方法を詳しく解説します。**Slidevは開発者のためのMarkdownベースプレゼンテーションツールで、従来のPowerPointでは困難だった技術発表の課題を解決します。**実務経験3年のエンジニアなら、既存のWeb開発スキルを活用して5日間で実戦レベルまで習得可能です。

Vue.jsとViteの最新技術スタックを基盤とし、Markdownでコンテンツを記述しながらインタラクティブな要素を組み込める点が最大の特徴です。コードのシンタックスハイライト、ライブコーディング、図表の動的生成など、技術発表に特化した機能が豊富に搭載されています。本ガイドでは、効率的な学習ステップから実践的な活用方法まで、プロダクションレベルでの使用に必要な全要素を網羅的に解説します。

## Slidevとは：開発者のための次世代プレゼンテーションツール

**Slidevは「slide + dev」を組み合わせた造語で、開発者専用に設計されたWebベースのプレゼンテーション作成ツールです。**従来のPowerPointやGoogle Slidesとは根本的に異なり、MarkdownとVue.jsを組み合わせてスライドを作成します。Anthony Fu氏によって開発され、GitHub上で38,000以上のスターを獲得している注目のオープンソースプロジェクトです。

技術スタックの中核には**Vite**（高速な開発サーバー）、**Vue 3**（リアクティブなコンポーネントシステム）、**UnoCSS**（ユーティリティファーストCSS）、**Shiki**（シンタックスハイライト）が組み込まれています。この組み合わせにより、瞬時のホットリロード、リアルタイムプレビュー、コンポーネントベースの再利用性を実現しています。

### 他ツールとの決定的な違い

**PowerPoint/Google Slidesとの比較**では、Slidevの優位性は明確です。コードスニペットが第一級市民として扱われ、ライブコーディング環境が標準搭載されています。バージョン管理（Git）との完全な統合により、プレゼンテーションの履歴管理と共同編集が可能です。一方、PowerPointは直感的なWYSIWYG編集が可能ですが、技術的なコンテンツの表現力に限界があります。

**Reveal.jsとの比較**では、SlidevはMarkdownベースでより簡潔な記述が可能です。Vue.jsサポートによりインタラクティブなコンポーネントが標準で利用でき、ViteベースのHMRで開発体験が格段に向上しています。Reveal.jsはHTML組織化が必要で、プラグイン依存度が高いという課題があります。

**Marp**と比較すると、同様のMarkdownベースでありながら、Slidevは拡張性と機能性で大きく上回ります。インタラクティブコンポーネント、ライブコーディング、高度なアニメーション機能を標準搭載し、テーマエコシステムも充実しています。

## 技術発表での圧倒的なメリット

**コード中心の技術発表において、Slidevは従来ツールでは不可能だった表現力を提供します。**Shikiによる100以上の言語対応シンタックスハイライト、行単位でのコード強調表示、Magic Move機能によるコード変遷のアニメーション表示が可能です。Monaco Editorの統合により、プレゼンテーション中にリアルタイムでコードを実行・デバッグできます。

**ダイアグラムとドキュメンテーションの統合**も大きな強みです。MermaidやPlantUMLによるテキストベースの図表作成、KaTeXによる数式表示、カスタムVueコンポーネントによる専門的な可視化が組み込まれています。これにより、アーキテクチャ図、フローチャート、API仕様書などを動的に生成・表示できます。

**開発ワークフローとの完全統合**により、プレゼンテーション作成が開発プロセスの自然な延長となります。Gitによるバージョン管理、CLIツールによる自動化、CI/CDパイプラインでの自動デプロイが可能です。既存の開発環境やツールチェーンをそのまま活用できるため、学習コストを最小限に抑えながら高品質なプレゼンテーションを作成できます。

## 5日間完全習得プラン：実務レベルまでの最短ルート

### 前提知識とスキル要件

**必須スキル**として、Node.js 18.0以上の環境構築、基本的なMarkdown記法、HTML/CSSの基礎知識、コマンドライン操作、Gitの基本的な使用法が必要です。**強く推奨されるスキル**には、JavaScript ES6+の理解、ユーティリティファーストCSS（Tailwind/UnoCSS）の概念、YAMLの設定ファイル記法、Vue.jsのコンポーネント概念があります。

**学習加速要因**として、Viteビルドツールの経験、TypeScriptによる型安全な開発、pnpmパッケージマネージャーの使用があげられます。実務3年のWebエンジニアであれば、これらの大部分は既に習得済みと想定されるため、Slidev固有の機能に集中して学習を進められます。

### Day 1: 基盤構築とセットアップ（3-4時間）

**午前セッション**では環境構築から始めます。`pnpm create slidev`でプロジェクトを作成し、デフォルトのプロジェクト構造とデモスライドを理解します。基本的なMarkdownスライド記法、スライド区切り（`---`）、フロントマターの設定方法を習得します。

**実践課題**：技術的な話題で5枚のスライド作成（タイトル、箇条書き、コードブロック、画像、結論）を行い、ライブリロードワークフロー（`npm run dev`）に慣れます。

**午後セッション**では、プレゼンターノート記法（`<!-- notes -->`）、プレゼンターモードとナビゲーション制御、組み込みレイアウト（default、center、cover、image-right等）の使い分け、YAMLフロントマターによる詳細設定を学習します。

### Day 2: スタイリングとレイアウトマスタリー（3-4時間）

**UnoCSS**のユーティリティクラスを駆使したレスポンシブデザイン、テーマシステムの理解と選択、フォントとグローバルスタイリングの設定を学習します。`layouts/`ディレクトリでのカスタムレイアウト作成、コンポーネントスロットシステム、`styles/`ディレクトリでのCSS カスタマイゼーションも習得します。

**成果物**：前日の基本プレゼンテーションをプロフェッショナルなデザインに再構築し、一貫した ビジュアルテーマを適用します。

### Day 3: インタラクティブコンポーネントとコード表現（3-4時間）

Vue.jsコンポーネントの統合、`components/`ディレクトリでのカスタムコンポーネント作成、クリックアニメーション（`v-click`、`v-after`、`v-motion`）、コードハイライトと段階的な行表示を習得します。Monaco Editorを使用したライブコーディング例、図表統合（Mermaid、PlantUML）、Iconifyアイコンの使用法、組み込みコンポーネント（Tweet、YouTube等）の活用も学習します。

### Day 4: 高度な機能とカスタマイゼーション（3-4時間）

カスタムテーマの作成とブランディング、描画・注釈機能（Drauu統合）、録画・カメラビュー機能、プラグインシステムとVite統合、高度なUnoCSS設定とショートカットを習得します。マルチファイルプレゼンテーション構造、ビルド設定と最適化、Git統合による共同作業ワークフローも学習します。

### Day 5: プロダクション運用と展開（3-4時間）

**エクスポートオプション**（PDF、PPTX、PNG、SPA）のマスタリー、デプロイメント戦略（GitHub Pages、Netlify、Vercel）、プレゼンテーション最適化とパフォーマンス調整、SEOメタデータと共有オプションの設定を学習します。将来使用のためのプレゼンテーションテンプレート作成、カスタムコンポーネントライブラリの文書化、再利用可能なテーマシステムの構築も含まれます。

## セットアップと重要機能の詳細

### 開発環境セットアップ手順

**推奨インストール方法**：
```bash
# pnpm使用（推奨・高速）
npm i -g pnpm
pnpm create slidev

# npm使用
npm init slidev@latest

# Docker使用
docker run --name slidev --rm -it \
  --user node \
  -v ${PWD}:/slidev \
  -p 3030:3030 \
  tangramor/slidev:latest
```

**プロジェクト構造**は`components/`（カスタムVueコンポーネント）、`layouts/`（カスタムスライドレイアウト）、`public/`（静的アセット）、`setup/`（カスタムセットアップ）、`styles/`（カスタムスタイル）、`slides.md`（メインプレゼンテーションファイル）で構成されます。

### よく使われる機能と設定

**コア機能**として、Markdownベーススライド、Vueコンポーネント統合、Shikiによるコードハイライト、Monaco Editorによるライブコーディング、図表サポート（Mermaid、PlantUML）、KaTeXによる数式表示、描画・注釈ツール、プレゼンターモード、録画機能、複数形式エクスポートがあります。

**人気レイアウト**には、`cover`（タイトルスライド）、`default`（標準コンテンツ）、`two-cols`（2列レイアウト）、`image-right/left`（画像付きコンテンツ）、`section`（セクション区切り）、`fact`（重要事実の強調）、`quote`（引用）があります。

## ベストプラクティスと注意点

### 効果的なプレゼンテーション作成のベストプラクティス

**コンテンツ構成**では、マルチエントリー機能を使用した論理的セクション分割、Markdownの簡潔性を活用したコンテンツ重視のアプローチ、カスタム作成前の組み込みレイアウトシステムの積極活用が重要です。

**コードプレゼンテーション**では、`{7,13|5|6|9-12|*}`記法による段階的コード説明、Magic Move機能によるスムーズなコード変遷アニメーション、`v-clicks`とコードブロックの同期表示が効果的です。

**視覚デザイン**では、会場の照明条件を考慮したライト・ダークモード対応テーマの選択、UnoCSS Iconプリセットによる一貫したビジュアル要素、CSS GridとFlexboxによるレスポンシブレイアウトが推奨されます。

### 学習時のつまずきやすいポイントと対策

**インストールと設定の問題**として、グローバルインストールではなくローカルプロジェクト設定の使用、単一パッケージマネージャーの一貫した使用、Node.js 18.0以上の確実なインストールが重要です。

**コンテンツと構造の間違い**では、カスタムレイアウト作成前の組み込みレイアウトの習得、画像パスの絶対パス使用によるビルドエラー回避、過度なスライドトランジションによる注意散漫の防止が必要です。

**パフォーマンスアンチパターン**として、大容量画像ファイルの事前最適化、必要なテーマとアドオンのみのインストール、レンダリング問題を避けるためのシンプルなカスタムVueコンポーネント設計が重要です。

## 実践的な開発ワークフローと統合

### VS Code統合と効率的開発

**Slidev拡張機能**により、スライドツリービュー、プレビューパネル、開発サーバー管理が可能になります。パッケージマネージャー設定のカスタマイズ（`slidev.dev-command`）、ワークスペース機能による複数Slidevプロジェクトの管理も効率化に寄与します。

**開発プロセス**では、ViteのHMRによる瞬時アップデート、別ファイルでのセクション作成による保守性向上とコラボレーション促進、Git対応Markdown形式による適切なバージョン管理と共同作業が可能です。

### デプロイメントと運用最適化

**ビルド最適化**では、特定スライドエクスポート（`--range`フラグ）、目的別エクスポート形式選択（共有用PDF、画像用PNG、PowerPoint互換PPTX）、サブルートデプロイメント用ベースパス設定（`--base`オプション）が重要です。

**プレゼンテーション配信**では、オフラインプレゼンテーション用静的SPA、セカンダリディスプレイでのプレゼンターモード設定、必要時の描画ツール有効化によるインタラクティブプレゼンテーションが可能です。

このガイドに従って学習を進めることで、実務3年のWebアプリケーションエンジニアは5日間でSlidevを実戦レベルまで習得し、技術発表の品質と効率性を大幅に向上させることができます。
